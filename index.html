Worst-1% Scene Similarity

For each of the 6 worst-1% val samples (highest per-sample prediction loss), the most similar val scenes retrieved by 3 independent methods. 5-level stratified retrieval (level 1 = most similar → level 5 = least), the query's own scene self-excluded.

Method 1 — Trajectory-latent similarity

GT future-trajectory velocity (per-channel whitened) encoded by a frozen autoencoder (latent_dim=4), cosine similarity. Query = the exact worst window (by ds_idx); corpus = full val, scene-level (6968 scenes, one window each). No model prediction involved — pure trajectory dynamics. Query captions show loss / ADE / speed.

Methods 2 & 3 — SAM3 video similarity (DINOv2)

SAM3-masked 4-frame past trajectory (frames 0/4/8/12) → DINOv2 dinov2_vitb14 forward_features, per-frame L2-normalized then averaged. Method H = averaged CLS token, Method J = averaged mean-patch token. Corpus = 1006 val scenes (the 6 queries + 1000 random), freshly segmented with SAM3. Four mask classes; "combined" = road + vehicle + pedestrian embedded separately and concatenated (2304-dim). Each report uses the same layout and trajectory-clip cells as Method 1 — only the retrieval metric differs, so the methods can be compared scene-for-scene.

Method 4 — Late fusion (appearance + action)

Joint cosine = 0.5·appearance + 0.5·action, where appearance is the whitened SAM3/DINOv2 feature (top-2 common directions removed) and action is the Method-1 velocity-latent (GT future velocity → frozen autoencoder, dim-4). Retrieves scenes that look similar AND move similarly, so a turning query no longer matches straight-driving scenes. Same Method-1 layout.

Method 5 — Learned contrastive heads (frozen backbones)

A small MLP projection head trained on top of the frozen appearance feature so that appearance retrieval respects action — no action feature needed at query time. Option A (single tower) pulls together scenes with similar ego maneuver (action-defined positives). Option B (two-tower CLIP) aligns each scene's appearance embedding with its own action embedding, symmetric InfoNCE. Trained on the 1000 corpus scenes (6 queries held out), appearance = "combined" mask. (Reports generating — will appear here shortly, together with the SAM3 mask-overlay update across all pages.)
